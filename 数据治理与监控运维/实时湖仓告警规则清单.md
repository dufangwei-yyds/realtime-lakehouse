## 一、文档目标

本清单用于沉淀实时湖仓一期的告警规则，目标是：

1. 统一定义告警对象、触发条件、级别和默认动作
2. 为 Prometheus / Alertmanager / 钉钉等告警实现提供清晰台账
3. 为值守、巡检、发布验证、项目验收提供依据

本清单与其他文档的关系：

- 主体方案讲“告警体系怎么设计”
- 模板文档讲“规则模板和指标怎么写”
- 本清单讲“当前到底启用了哪些告警”

---

## 二、告警分级约定

| 级别 | 说明 | 默认处置要求 |
|---|---|---|
| `P0` | 业务主链路中断，核心结果不可用 | 立即人工介入，必要时暂停消费 |
| `P1` | 主结果严重异常，影响业务使用 | 立即处理，当日闭环 |
| `P2` | 运行风险高，短期可能影响主链路 | 当日处理 |
| `P3` | 轻度异常或趋势风险 | 纳入巡检与优化计划 |
| `P4` | 提示类或观测类 | 记录即可 |

## 三、告警字段说明

后续每条告警统一采用以下字段描述：

| 字段 | 说明 |
|---|---|
| 告警编号 | 唯一编号 |
| 告警名称 | 中文名称 |
| 告警级别 | `P0-P4` |
| 监控对象 | 作业、表、Topic、服务、结果集 |
| 触发条件 | 指标或规则条件 |
| 检测方式 | Prometheus / SQL / 页面检查 / 脚本 |
| 告警路由 | 运维 / 数据开发 / 业务接口人 / BI |
| 默认动作 | 告警触发后的标准动作 |
| 建议阈值 | 当前建议阈值 |

## 十六、告警落地建议

### 16.1 Prometheus 规则管理建议

- 按层级拆分规则文件：
  - `flink_common_alerts.yml`
  - `ods_alerts.yml`
  - `dwd_dws_ads_alerts.yml`
  - `doris_alerts.yml`

### 16.2 Alertmanager 路由建议

- 按级别分组
- 按责任角色分路由
- `P1/P0` 建议直接路由到即时沟通工具

### 16.3 阈值治理建议

- 初始阈值先保守
- 运行一段时间后根据噪音情况微调
- 所有阈值调整必须记录版本
