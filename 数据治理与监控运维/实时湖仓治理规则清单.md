## 一、文档目标

本清单用于将实时湖仓一期已经形成的治理要求，沉淀成“可执行、可核查、可验收”的规则台账，目标是：

1. 统一管理全链路治理规则
2. 明确每条规则的作用对象、校验方式、责任归属和处置方式
3. 为日常巡检、问题定位、项目验收提供依据

本清单与主体方案的关系是：

- 主体方案讲“治理框架和原则”
- 本清单讲“具体规则有哪些”

## 二、规则分层与分类

### 2.1 分层范围

| 层级 | 规则重点 |
|---|---|
| ODS | 贴源真实性、元数据完整性、分区正确性、接入时效 |
| DIM | 主数据唯一性、维度完整性、维表一致性 |
| DWD | 明细幂等、补维成功率、时间语义正确性 |
| DWS | 公共粒度、指标一致性、聚合结果稳定性 |
| ADS | 消费一致性、解释字段完整性、刷新时效 |

### 2.2 规则分类

| 分类 | 说明 |
|---|---|
| 标准规则 | 命名、字段、主键、分区、编码标准 |
| 质量规则 | 完整性、唯一性、及时性、一致性、合法性 |
| 运行规则 | 作业、存储、服务、消费层运行约束 |
| 变更规则 | 结构变更、口径变更、发布变更控制 |
| 安全规则 | 权限、敏感信息、环境隔离 |

## 十三、规则执行优先级建议

### 13.1 一级必执行规则

这些规则建议纳入每日值守：

- ODS 关键接入完整性
- DIM 主键唯一性
- DWD 主键幂等
- DWS 结果时效
- ADS 刷新时效
- Flink 作业存活
- Checkpoint 稳定
- Doris 服务可用

### 13.2 二级增强规则

这些规则建议纳入每周或每月治理：

- 数据字典一致性
- 指标口径复盘
- MV 命中率与清理
- Snapshot 增长趋势
- 配置漂移检查
