# 实时湖仓全链路数据质量与监控预警模板

> 文档定位：实时湖仓 `ODS -> DIM -> DWD -> DWS -> ADS` 全链路质量与监控预警模板  
> 适用场景：一期项目治理与监控运维体系建设、后续主题域复制推广

---

## 1. 模板目标

本模板用于将已在 ODS 层验证有效的质量与监控经验，抽象为一套可覆盖全链路的统一模板，目标是：

1. 统一质量治理维度
2. 统一监控指标设计
3. 统一告警分级与处置方式
4. 统一 Prometheus / Grafana / Alertmanager 接入思路
5. 为 ODS、DIM、DWD、DWS、ADS 提供同构治理框架

---

## 2. 总体设计原则

### 2.1 质量治理原则

- 先定义“什么叫健康”，再定义“如何监控”
- 先识别会破坏业务语义的问题，再决定是否升级为告警
- 先按层治理，再做跨层核对
- 质量规则必须可复用、可追溯、可验收

### 2.2 监控设计原则

- 业务指标与系统指标并重
- 结果监控优先于资源监控
- 异常趋势监控优先于瞬时值监控
- 告警要分级，不做“所有问题都打爆群”

### 2.3 处置原则

- P1 / P0 问题优先保障主链路恢复
- 质量可疑时，先暂停对外消费，再继续向下扩散
- 结论不可信时，禁止继续自动触发业务动作

## 3. 全链路质量治理框架


| 层级  | 核心对象                                   | 重点风险                      | 核心治理点                 |
| --- | -------------------------------------- | ------------------------- | --------------------- |
| ODS | CDC 源表、事件流表、Latest / History、Analyze 表 | 源头丢数、Schema 漂移、迟到、重复、分区错误 | 原始留存、元数据完整、解析成功率、接入时效 |
| DIM | 退化宽表、统一维度表                             | 主数据不一致、维度字段缺失、状态错误        | 主数据唯一性、补维完整性、宽表一致性    |
| DWD | 明细宽表                                   | 重复膨胀、Join 失败、时间语义错误       | 明细幂等、补维成功率、事件时间稳定性    |
| DWS | 汇总表、排行表、特征表                            | 聚合口径错误、窗口延迟、结果不稳定         | 公共粒度、指标一致性、时效性        |
| ADS | 看板表、接口表、MV                             | 消费结果异常、刷新失败、解释字段缺失        | 结果时效、消费一致性、可解释性、SLA   |

## 4. 质量维度模板

### 4.1 通用质量维度


| 质量维度 | 说明             | 常见检查点                       |
| ---- | -------------- | --------------------------- |
| 完整性  | 必填字段是否缺失       | 主键、事件时间、状态码、来源字段            |
| 唯一性  | 是否发生重复膨胀       | 主键重复、消息重复、排行重复              |
| 及时性  | 是否在 SLA 内到达和产出 | ODS 入湖延迟、DWS 产出延迟、ADS 刷新延迟  |
| 一致性  | 上下游结果是否可核对     | 源头与 ODS、DWD 与 DWS、DWS 与 ADS |
| 合法性  | 字段值是否在业务约束范围内  | 金额、数量、状态、枚举、时间先后关系          |
| 稳定性  | 作业或结果是否波动异常    | 作业失败率、解析失败率、结果波动率           |


### 4.2 分层质量模板

#### ODS 层

- 主键/业务键非空
- `_event_time` 与 `_ingest_time` 合理
- 分区按事件时间正确路由
- 重复消息可识别
- JSON 解析成功率可观测

#### DIM 层

- 主维度主键唯一
- 关键维度属性齐全
- 停用、不可售状态可识别
- 宽表字段与 Latest 源表一致

#### DWD 层

- 明细主键不重复
- 维度补齐成功率达标
- 时间语义稳定
- 下游关键字段不出现大面积空值

#### DWS 层

- 公共粒度明确且稳定
- 指标计算公式固定
- 汇总结果可与 DWD 抽样核对

#### ADS 层

- 看板和接口结果可核对
- 关键解释字段存在
- 结果刷新时间满足 SLA

## 5.1 作业级指标


| 指标                        | 含义             | 适用层级                  |
| ------------------------- | -------------- | --------------------- |
| `job_running_status`      | 作业运行状态         | ODS / DIM / DWD / DWS |
| `checkpoint_success_rate` | Checkpoint 成功率 | ODS / DIM / DWD / DWS |
| `checkpoint_duration`     | Checkpoint 耗时  | ODS / DIM / DWD / DWS |
| `backpressure_level`      | 背压等级           | ODS / DWD / DWS       |
| `processing_latency`      | 作业处理延迟         | ODS / DWD / DWS       |
| `input_rate`              | 输入吞吐           | ODS / DWD / DWS       |
| `output_rate`             | 输出吞吐           | ODS / DWD / DWS       |

### 5.1.1 ODS 维度 SCD2 作业级自定义业务指标

| 指标 | 含义 | 适用说明 |
|---|---|---|
| `scd2_input_total` | 输入数据量 | 进入 SCD2 作业的维度变更总量 |
| `scd2_output_total` | 新版本生成量 | 成功输出的新版本记录数量 |
| `scd2_nochange_total` | 无变化更新量 | 输入更新但未触发版本变化的数量 |
| `scd2_close_version_total` | 闭合版本量 | 成功将旧版本 `end_time` 闭合的数量 |
| `scd2_version_jump_total` | 版本跳跃异常量 | 时间线或版本序列异常数量 |
| `scd2_processing_latency_seconds` | SCD2 处理延迟 | 源头变更到结果输出的处理时长 |

> 这组指标对 ODS 维度层非常关键，因为它们直接决定：
>
> - SCD2 是否在正确产出新版本
> - SCD2 是否在正确闭合旧版本
> - 是否存在“输入了但没有变化”“版本跳跃”“延迟异常”等高风险问题

### 5.1.2 Flink 原生系统级指标

| 指标 | 含义 | 适用说明 |
|---|---|---|
| `flink_jobmanager_numRegisteredTaskManagers` | 注册 TM 数量 | 判断集群是否健康 |
| `flink_taskmanager_Status_JVM_CPU_Load` | JVM CPU 使用率 | 判断 TM 是否在正常工作 |
| `flink_taskmanager_job_task_busyTimeMsPerSecond` | busyTime | 判断任务是否背压 |
| `flink_jobmanager_job_checkpoint_*` | Checkpoint 系列指标 | 判断状态一致性与容错是否健康 |

### 5.1.3 ODS 维度 SCD2 存储健康度增强指标

| 指标 | 说明 |
|---|---|
| `paimon_sink_records_written` | 实际写入量 |
| `paimon_compaction_duration` | Compaction 耗时 |
| `paimon_compaction_pending` | Compaction 积压 |
| `paimon_snapshot_count` | Snapshot 增长趋势 |

> 指标获取建议：
>
> - 使用 Paimon Sink 内置 Metrics
> - 开启 `metrics.enabled`
> - 重点关注 `snapshotCount` 的趋势而不是单点值
>
> 典型趋势预警：
>
> - `increase(paimon_snapshot_count[1h]) > 100`

## 8. 告警分级模板


| 级别  | 说明          | 示例                                  | 默认动作           |
| --- | ----------- | ----------------------------------- | -------------- |
| P0  | 主链路中断且业务不可用 | 全链路 ADS 无结果、核心作业连续失败                | 立即人工介入，必要时暂停消费 |
| P1  | 主结果严重异常     | DWS / ADS 延迟严重超 SLA、主键膨胀、版本跳跃       | 立即定位与修复        |
| P2  | 运行风险高       | Compaction 积压、Kafka Lag 持续升高、MV 不刷新 | 当日处理           |
| P3  | 轻度异常        | 轻微延迟、轻微波动、趋势型容量风险                   | 记录并纳入巡检        |
| P4  | 提示类         | 例行提醒、观测类事件                          | 仅记录            |

## 9. Prometheus 告警规则模板

### 8.1 Flink + Paimon 通用模板

```yaml
groups:
  - name: realtime-lakehouse-common
    rules:
      - alert: FlinkCheckpointConsecutiveFailures
        expr: increase(flink_jobmanager_job_checkpoint_failures_total[5m]) >= 3
        for: 1m
        labels:
          severity: critical
          level: P1
        annotations:
          summary: "Flink Job {{ $labels.job }} 连续 Checkpoint 失败"

      - alert: FlinkTaskBackpressureHigh
        expr: max_over_time(flink_taskmanager_job_task_busyTimeMsPerSecond[2m]) / 1000 > 0.8
        for: 1m
        labels:
          severity: warning
          level: P2
        annotations:
          summary: "Flink Job {{ $labels.job }} 背压严重"

      - alert: PaimonCompactionBacklog
        expr: paimon_compaction_pending > 5
        for: 10m
        labels:
          severity: warning
          level: P2
        annotations:
          summary: "Paimon Compaction 持续积压"

      - alert: PaimonSnapshotGrowthAbnormal
        expr: increase(paimon_snapshot_count[1h]) > 100
        for: 10m
        labels:
          severity: warning
          level: P3
        annotations:
          summary: "Paimon Snapshot 增长异常"
```

## 10. ODS 维度 SCD2 集成点模板

### 10.1 集成建议

对于当前使用 Flink DataStream API 实现的维度 SCD2 作业，建议在以下位置埋点：

- `Scd2ProductJob`：注册作业级指标与统一命名空间
- `Scd2ProductProcessFunction`：输入计数、无变化计数、闭合计数、版本跳跃计数
- `Scd2ProductProcessFunctionV2`：兼容增强版本逻辑的额外指标
- `Scd2OutputRecord`：输出阶段指标补齐

### 10.2 集成点示例

```java
stream
    .keyBy(productId)
    .flatMap(new Scd2ProductProcessFunction())
    .sinkTo(paimonSink);
```

设计建议：

- 不侵入核心业务语义
- 指标注册放在作业初始化阶段
- 指标更新放在 `processElement` 和输出分支
- 异常分支单独打点，避免与正常输出混淆

### 10.3 这部分在生产中的真正价值

| 传统问题 | 现在 |
|---|---|
| SCD2 是否正常不知道 | Dashboard 秒级可见 |
| Paimon 表无声膨胀 | Compaction 指标提前预警 |
| Checkpoint 事故靠人工发现 | 可自动预警 |
| 运维靠经验 | 运维靠指标 |

## 11. 巡检模板

### 9.1 每日巡检模板


| 检查项             | 关注点                    | 适用层级                  |
| --------------- | ---------------------- | --------------------- |
| 作业是否运行          | 主作业是否全部存活              | 全链路                   |
| Checkpoint 是否成功 | 是否连续失败或耗时过长            | ODS / DIM / DWD / DWS |
| Kafka 是否积压      | Topic Lag 是否异常         | ODS                   |
| Paimon 是否膨胀     | 快照和小文件是否异常             | ODS / DIM / DWD       |
| Doris 是否健康      | FE / BE 是否正常、MV 是否刷新   | DWS / ADS             |
| 结果是否按时更新        | DWS / ADS 更新时间是否满足 SLA | DWS / ADS             |


### 9.2 异常巡检模板


| 异常现象     | 优先排查                             |
| -------- | -------------------------------- |
| ADS 无结果  | DWS -> DWD -> ODS 逐层回查           |
| DWS 延迟   | Flink 窗口作业、Doris Sink、Checkpoint |
| DWD 结果异常 | DIM 补维、Join 命中率、主键重复             |
| ODS 条数异常 | Kafka Lag、CDC 延迟、解析失败率           |

## 12. 健康评分模板

### 10.1 Paimon 健康评分


| 维度            | 规则                                   | 扣分  |
| ------------- | ------------------------------------ | --- |
| Snapshot 增长   | `increase(snapshot_count[1h]) > 100` | -20 |
| Compaction 积压 | `compaction_pending > 5`             | -30 |
| 无 Compaction  | 超过 10 分钟未触发                          | -20 |
| Snapshot 失败   | `snapshot_failures > 0`              | -40 |



| Score   | 状态        | 说明          |
| ------- | --------- | ----------- |
| `>= 90` | Healthy   | 可放心对账       |
| `70-89` | Degraded  | 需观察、必要时延后对账 |
| `< 70`  | Unhealthy | 禁止直接对外使用    |


### 10.2 全链路健康评分建议


| 层级  | 评分维度               |
| --- | ------------------ |
| ODS | 接入延迟、解析成功率、重复率     |
| DIM | 主数据完整性、宽表成功率       |
| DWD | 补维成功率、主键重复率        |
| DWS | 结果时效、聚合稳定性         |
| ADS | 刷新时效、接口 SLA、MV 健康度 |
