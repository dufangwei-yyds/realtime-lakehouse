# Doris AI测试数据方案

> 文档定位：`Doris 4.x` AI 功能实验测试数据方案  
> 适用阶段：`Doris AI` 功能实验第一轮  
> 目标：为 `AI Functions / Text Search / Vector Search / 工单文本增强` 等实验提供统一、可解释、可复用的测试数据底稿

---

## 一、方案目标

本方案服务于 [Doris AI功能实验规划.md](/d:/workspace/realtime-lakehouse/Doris%20AI功能实验规划.md) 中首轮建议纳入的 5 个实验项：

1. 社媒情感分析
2. 社媒内容摘要
3. 商品文本全文检索
4. 商品向量检索
5. 工单/投诉分类与脱敏

本轮测试数据设计强调：

- 业务语义清晰
- 结果易于人工判断
- 便于在 Superset / SQL / 文档中展示
- 便于后续实验记录和结论沉淀

---

## 二、设计原则

### 2.1 服装行业语义优先

数据内容优先围绕服装行业真实语境设计，例如：

- 商品名称
- 商品卖点
- 社媒笔记文案
- 用户评价
- 客服投诉

### 2.2 可解释优先

每一条样例尽量都应“人眼一看就知道期望结果”，方便校验：

- 正向/负向情绪
- 摘要是否合理
- 搜索是否命中
- 分类是否合理
- 脱敏是否有效

### 2.3 先小后大

第一轮不追求海量数据，而是优先使用“小而精”的样例集跑通能力。

### 2.4 为后续扩展留口

首轮样例集虽然不大，但字段设计要兼顾后续扩展，例如：

- 加 embedding 列
- 加标签列
- 加业务主题列

---

## 三、建议的数据对象

本轮建议准备 4 组核心测试数据表。

### 3.1 社媒文本实验表

表名建议：

- `ai_social_note_exp`

用途：

- `AI_SENTIMENT`
- `AI_SUMMARIZE`
- `AI_CLASSIFY`
- `Text Search`
- 后续可扩展到 `Vector Search`

### 3.2 商品知识实验表

表名建议：

- `ai_product_knowledge_exp`

用途：

- `Text Search`
- `Vector Search`
- `AI_EXTRACT`
- `AI_TRANSLATE`

### 3.3 客服工单实验表

表名建议：

- `ai_service_ticket_exp`

用途：

- `AI_CLASSIFY`
- `AI_MASK`
- `AI_SUMMARIZE`

### 3.4 商品向量实验表

表名建议：

- `ai_product_vector_exp`

用途：

- `Vector Search`

说明：

- 第一轮若暂不准备真实 embedding，可先保留结构
- 后续可基于已有实践文档补充向量样例

## 四、社媒文本实验数据设计

| `sku_id` | BIGINT | 关联商品 SKU |

## 八、推荐执行顺序

建议按以下顺序准备和使用测试数据：

1. 先准备社媒文本实验数据
2. 再准备商品知识实验数据
3. 再准备客服工单实验数据
4. 最后补商品向量实验数据

原因：

- 社媒文本和商品知识最贴当前实验优先级
- 客服工单稍后补即可
- 向量检索依赖稍多，适合放后面
