## 二、组件选型说明

### 2.1 Flink CDC

**适用场景**
- MySQL、PostgreSQL 等关系型业务库增量接入
- ERP 主数据（商品、门店、用户）
- 订单主表和订单明细
- 库存流水

**使用原则**
- ? 使用 Flink CDC 2.4.x 版本
- ? 优先使用 Debezium JSON 格式
- ? 必须配置 scan.startup.mode（initial/latest-offset）
- ? 禁止使用全量模式处理增量场景

**典型配置**
```sql
CREATE TEMPORARY TABLE cdc_source (
    ...
    PRIMARY KEY (id) NOT ENFORCED
) WITH (
    'connector' = 'mysql-cdc',
    'hostname' = '${host}',
    'port' = '3306',
    'username' = '${user}',
    'password' = '${pwd}',
    'database-name' = '${db}',
    'table-name' = '${table}',
    'scan.startup.mode' = 'initial'
);
```

### 2.2 Kafka

**适用场景**
- 行为埋点数据接入
- 物流轨迹事件接入
- 社媒舆情事件接入
- API/爬虫采集结果缓冲

**使用原则**
- ? 使用 Kafka 作为统一事件总线
- ? 主题命名遵循 `rt_{domain}_{object}_{event}` 规范
- ? 配置 json.ignore-parse-errors='true' 容错
- ? 必须声明 WATERMARK 策略

**典型配置**
```sql
CREATE TEMPORARY TABLE kafka_source (
    ...
    WATERMARK FOR event_time AS event_time - INTERVAL '1' MINUTE
) WITH (
    'connector' = 'kafka',
    'topic' = '${topic}',
    'properties.bootstrap.servers' = '${servers}',
    'format' = 'json',
    'json.ignore-parse-errors' = 'true'
);
```

### 2.3 Flink SQL

**适用场景**
- ODS 标准化接入
- DIM 宽表建设
- DWD 明细打宽
- DWS 主题聚合

**使用原则**
- ? 主体实时建模优先使用 Flink SQL
- ? 窗口作业优先使用单语句 WITH...SELECT... 模式
- ? CREATE TABLE 必须补齐字段注释和关键属性
- ? 复杂状态控制场景不使用 Flink SQL

### 2.4 DataStream API

**适用场景**
- ODS 维度 SCD2 专项处理
- 复杂状态控制逻辑
- 复杂定制指标输出

**使用原则**
- ? 仅在 Flink SQL 无法满足时使用
- ? 自定义业务指标必须接入监控
- ? 状态逻辑必须可解释
- ? 关键 ProcessFunction 逻辑需有注释

### 2.5 Paimon

**适用场景**
- ODS 层原始数据沉淀
- DIM 层维度存储
- DWD 层明细沉淀
- DWS 层汇总结果沉淀

**使用原则**
- ? 使用 Paimon 作为湖仓主存储
- ? Append-Only 表用于纯事件流
- ? 主键表用于需要唯一性的场景
- ? 必须定义 WATERMARK

**存储引擎选择**

| 场景 | 写入模式 | Merge Engine | 说明 |
|-----|---------|--------------|------|
| 行为事件流 | append-only | - | 高吞吐，只追加 |
| 订单/库存事实 | 主键表 | deduplicate | 去重合并 |
| SCD2 历史 | 主键表 | aggregate | 保留历史 |
| 维度拉链表 | 主键表 | partial-update | 拉链存储 |

### 2.6 Doris

**适用场景**
- ADS 结果集存储
- Dashboard 高并发查询
- API 接口查询
- 物化视图加速

**使用原则**
- ? ADS 表优先面向消费对象设计
- ? 高频结果考虑物化视图
- ? 单 BE 测试环境指定 replication_num=1

**外部表配置**
```sql
CREATE TABLE ads_external (
    sku_id VARCHAR(64),
    sku_name VARCHAR(255),
    sale_amt DECIMAL(18,2)
) ENGINE=JDBC
PROPERTIES (
    "resource" = "paimon_resource",
    "table" = "dwd_order_detail_rt"
);
```

## 三、技术选型原则

### 3.1 五大原则

| 原则 | 说明 | 一期实践 |
|-----|------|---------|
| 稳定优先 | 选择生态成熟、与团队能力匹配的组件 | Flink CDC + Kafka + Flink 组合稳定 |
| 分层解耦 | 接入、计算、存储、查询、消费职责清晰 | ODS/DIM/DWD/DWS/ADS 分层明确 |
| 实时与可维护并重 | 便于排障、监控、回滚和持续维护 | 统一元数据、可监控可回滚 |
| 明细与服务并重 | 既要沉淀明细，也要对外服务 | Paimon 存明细，Doris 做服务 |
| 预留演进空间 | 一期稳定，二期扩展 | 预留 Fluss 和 Doris AI 方向 |

### 3.2 替代方案对比

**方案对比：Kafka+Flink+Doris vs Kafka+Flink+Paimon+Doris**

| 维度 | 纯 Doris 方案 | Paimon+Doris 方案 |
|-----|--------------|-------------------|
| 链路复杂度 | 简单 | 略复杂 |
| 明细沉淀能力 | 弱 | 强 |
| 可重算基础 | 无 | 有 |
| 适合场景 | 轻量实时 | 明细+查询并重 |

? **一期选型：Paimon + Doris 双引擎协同**

## 四、数据流架构

### 4.1 核心数据流

```
业务库(MySQL) ──CDC──> Kafka ──Flink──> Paimon(ODS)
                                          │
                                          ▼
                                    Paimon(DIM)
                                          │
                                          ▼
                                    Paimon(DWD)
                                          │
                                          ▼
                                    Paimon(DWS)
                                          │
                                          ▼
                              ┌───────────┴───────────┐
                              ▼                       ▼
                       Paimon(ADS)              Doris(ADS)
                              │                       │
                              └───────────┬───────────┘
                                          ▼
                                    Superset/Dashboard/API
```

### 4.2 场景与技术匹配

| 业务场景 | 关键数据 | 技术支撑 |
|---------|---------|---------|
| 全渠道实时库存可视与智能调拨 | 订单、库存、维度主数据 | Flink CDC + Flink SQL + Paimon + Doris |
| 爆款趋势识别与快速追单 | 行为、社媒、订单、库存 | Kafka + Flink SQL + Paimon + Doris |

## 五、后续演进方向

### 5.1 Apache Fluss（预研阶段）

**定位**：二阶段轻量预研方向，不替代一期主链路

**建议验证方向**
- 实时特征层
- 热点明细层
- 与 Flink 集成方式
- 吞吐与低延迟表现

### 5.2 Doris AI（实验阶段）

**定位**：Doris 4.x 增强能力方向，先做轻量实验

**建议验证方向**
- AI Function
- Text Search
- Vector Search
- 与商品、社媒、热点分析的结合可能性

## 六、配置参数参考

### 6.1 Flink 运行时配置

| 参数 | 推荐值 | 说明 |
|-----|-------|------|
| execution.checkpointing.interval | 10s | 检查点间隔 |
| execution.checkpointing.mode | EXACTLY_ONCE | 检查点模式 |
| execution.checkpointing.timeout | 10min | 检查点超时 |
| state.backend | rocksdb | 状态后端 |
| table.exec.state.ttl | 7d | 状态 TTL |

### 6.2 Paimon 表配置

| 参数 | 推荐值 | 说明 |
|-----|-------|------|
| bucket | 4 | 主键表分桶数 |
| write-mode | append-only/deduplicate | 写入模式 |
| merge-engine | deduplicate/partial-update/aggregation | 合并引擎 |
| file.format | orc | 文件格式 |

### 6.3 Kafka 消费者配置

| 参数 | 推荐值 | 说明 |
|-----|-------|------|
| scan.startup.mode | latest-offset | 启动模式（生产用 latest） |
| properties.auto.offset.reset | latest | 偏移量重置策略 |
| json.ignore-parse-errors | true | JSON 容错 |
