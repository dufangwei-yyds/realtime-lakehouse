# 实时湖仓开发规范 · 总纲

> 项目名称：服装行业实时湖仓构建项目一期  
> 文档定位：AI 行为总纲，定义实时湖仓开发的核心原则与边界  
> 适用范围：ODS -> DIM -> DWD -> DWS -> ADS 全链路开发、Flink 作业设计、数据存储选型

---

## 一、AI 生成原则

### 1.1 首要原则

? **所有开发决策必须优先查阅本规范文件体系**

- 在生成任何 Flink SQL 作业前，必须先读取 03-layer-modeling.md 确认分层边界
- 在设计任何 Paimon 表前，必须先读取 02-naming.md 和 08-paimon.md 确认命名和存储规范
- 在选择技术组件前，必须先读取 01-tech-stack.md 确认技术选型

### 1.2 禁止原则

? **以下情况严格禁止，不得出现在任何生成结果中**

| 禁止项 | 说明 |
|-------|------|
| 未定义的颜色值/类型 | 实时湖仓开发中指：未在 04-field.md 中定义的字段类型 |
| 未定义的组件变体 | 实时湖仓开发中指：未在 06-flink-sql.md 中列出的作业结构 |
| 非规范命名 | 未在 02-naming.md 中定义的表名、作业名、Topic 名 |
| 违规时间字段 | 使用 TIMESTAMP(0)、DATETIME 等替代 TIMESTAMP(3) |
| 缺失元数据字段 | ODS/DWD 层写入时不注入 _event_time、_ingest_time、_partition_dt |
| 跳过幂等处理 | 事实表写入时不考虑重复数据处理 |

### 1.3 生成优先级

1. **优先复用** - 参照 03-layer-modeling.md 中已有的分层模式
2. **其次组合** - 通过标准组件（CDC Source、Kafka Source、Paimon Sink）组合实现
3. **最后新建** - 仅在标准模式无法满足时新建，且必须主动说明原因

## 二、技术风格锚点

### 2.1 技术栈关键词

| 组件 | 关键词 | 说明 |
|-----|-------|------|
| Flink CDC | 增量接入、变更捕获 | 用于 MySQL 等关系型数据库接入 |
| Kafka | 事件总线、削峰缓冲 | 用于行为埋点、物流轨迹、社媒事件 |
| Flink SQL | 声明式、批量建模 | 主体实时建模首选 |
| DataStream API | 状态管理、SCD2 | 仅用于复杂状态控制场景 |
| Paimon | 流批一体、湖仓存储 | 主存储底座 |
| Doris | 交互查询、物化视图 | ADS 结果服务 |

### 2.2 架构风格

- ? 明确分层：ODS(贴源) -> DIM(维度) -> DWD(明细) -> DWS(汇总) -> ADS(应用)
- ? 统一总线：Kafka 作为事件流统一总线
- ? 湖仓一体：Paimon 承载明细沉淀，Doris 承载查询服务
- ? 禁止：过早宽表化、跳过 ODS 直连 DIM、混淆分层职责

### 2.3 技术选型原则

来自 01-tech-stack.md 的技术选型原则：

1. **稳定优先** - 优先选择生态成熟、与团队能力匹配的组件
2. **分层解耦** - 接入、计算、存储、查询、消费职责清晰
3. **实时与可维护并重** - 便于排障、监控、回滚和持续维护
4. **明细与服务并重** - 既要沉淀明细，也要对外服务
5. **预留演进空间** - 一期稳定，二期扩展 Fluss 和 Doris AI

## 三、实时湖仓核心概念

### 3.1 五层架构职责

| 层级 | 职责 | 核心特征 | 禁止事项 |
|-----|------|---------|---------|
| ODS | 贴源接入、轻度清洗、元数据注入 | 保持原始、保留漂移、快速入湖 | 重度业务聚合、过早宽表化 |
| DIM | 统一维度口径、维度退化 | append/history/latest 三态 | 维度主键不一致 |
| DWD | 事实打宽、维度关联、明细沉淀 | 主键唯一、事件时间统一 | 多业务过程混装 |
| DWS | 公共粒度、指标固化 | 粒度清晰、指标复用 | 口径不清、粒度混乱 |
| ADS | 面向消费、结果输出 | 解释字段、建议字段 | 重复底层公共指标 |

### 3.2 核心技术约束

| 约束项 | 规范值 | 说明 |
|-------|-------|------|
| 时间字段精度 | TIMESTAMP(3) | 毫秒级精度，禁止使用其他精度 |
| 金额字段类型 | DECIMAL(18,2) | 18位总精度，2位小数 |
| 数量字段类型 | DECIMAL(12,2) | 12位总精度，2位小数 |
| 分区字段类型 | STRING | 使用 yyyyMMdd 格式 |
| 主键跨层一致性 | 必须一致 | 主键类型和语义不得跨层变更 |

### 3.3 事件时间优先原则

? **所有统计与窗口计算必须使用 event_time（业务事件时间）**

? **严格禁止使用处理时间（processing time）代替业务事件时间**

## 四、符号约定与速查

### 4.1 符号含义

| 符号 | 含义 | 使用场景 |
|-----|------|---------|
| ? | 强制要求 | 必须在生成结果中体现 |
| ? | 严格禁止 | 不得出现在任何生成结果中 |
| ? | 最佳实践 | 推荐遵守，可根据场景调整 |
| ?? | 风险提示 | 需要特别注意的边界场景 |

### 4.2 核心术语速查

| 术语 | 含义 |
|-----|------|
| CDC | Change Data Capture，变更数据捕获 |
| Watermark | 水位线，用于处理乱序和迟到数据 |
| SCD2 | Slowly Changing Dimension Type 2，缓慢变化维度类型2（历史快照） |
| Append-Only | 仅追加模式，适合事件流写入 |
| Deduplicate | 去重合并引擎 |
| Grain | 粒度，数据汇总的最小维度组合 |

### 4.3 元数据字段速查

| 字段名 | 含义 | 注入层级 |
|-------|------|---------|
| _event_time | 业务事件时间 | ODS/DWD/DWS |
| _ingest_time | 入湖/入层时间 | ODS |
| _partition_dt | 日期分区（yyyyMMdd） | ODS/DWD/DWS |
| _partition_hh | 小时分区（HH） | ODS（行为流） |
| _kafka_offset | Kafka 消息偏移量 | ODS（Kafka 接入） |
| _kafka_partition | Kafka 分区号 | ODS（Kafka 接入） |

## 五、分层决策树

### 5.1 作业类型决策

```
问：这个作业做什么？
├── 接入业务库增量数据？
│   └── 是 → Flink CDC + ODS 层
├── 清洗/解析事件流？
│   └── 是 → Kafka Source + ODS 层
├── 统一维度口径/构建宽表？
│   └── 是 → DIM 层
├── 事实打宽/维度关联？
│   └── 是 → DWD 层
├── 公共粒度聚合/指标固化？
│   └── 是 → DWS 层
└── 面向 Dashboard/API 输出？
    └── 是 → ADS 层
```

### 5.2 表类型决策

```
问：这个表是什么类型？
├── 需要保留历史快照？
│   ├── 最新镜像 → ods_{table}_latest / dim_{domain}_wide
│   └── 完整历史 → ods_{table}_history / DWD 历史事实表
├── 纯事件流追加？
│   └── 是 → ods_{domain}_{event}_analyze
├── 需要主键唯一？
│   └── 是 → Paimon 主键表（merge-engine=deduplicate）
└── 高吞吐只追加？
    └── 是 → Paimon Append 表
```

### 5.3 时间字段决策

```
问：用什么作为事件时间？
├── 有明确的业务发生时间？
│   └── 是 → 使用业务时间作为 _event_time
├── 仅有 Flink 处理时间？
│   └── ?? 警告：必须使用业务时间，询问业务方或数据来源
└── 时间字段精度不足？
    └── ?? 警告：必须使用 TIMESTAMP(3)，不得降级
```

## 六、规范自检清单

### 6.1 所有任务必须检查

- [ ] 表名/作业名符合 02-naming.md 命名规范
- [ ] 时间字段使用 TIMESTAMP(3)
- [ ] 分区字段使用 STRING 类型，格式为 yyyyMMdd
- [ ] 已在对应分层规范中确认作业/表类型

### 6.2 ODS 层额外检查

- [ ] 已注入 _event_time、_ingest_time
- [ ] 行为流已注入 _partition_hh
- [ ] Kafka Source 已声明 WATERMARK
- [ ] JSON 解析使用 json.ignore-parse-errors='true'

### 6.3 DIM/DWD 层额外检查

- [ ] 维度关联使用 FOR SYSTEM_TIME AS OF 语法
- [ ] 主键已在文档中明确记录
- [ ] 考虑了重复数据的二次防护

### 6.4 DWS/ADS 层额外检查

- [ ] 粒度已在文档中明确
- [ ] 指标口径已定义
- [ ] 窗口/水位策略已明确
