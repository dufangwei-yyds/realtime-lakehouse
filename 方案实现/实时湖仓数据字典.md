## 一、编制说明

本字典是“一期全链路联调”前的第一版实时湖仓数据字典，目标是：

1. 统一梳理当前已实现的全链路对象清单
2. 明确各层创建了哪些表、视图、临时源表
3. 形成后续“批量生成源头测试数据”和“ODS 到 ADS 全链路联调”的统一参照

本版优先覆盖当前主线实现对象：

- ODS：CDC 源表、事件流源表、维度 `append/history/latest` 表、事实表、`analyze` 解析表
- DIM：商品退化维度宽表
- DWD：订单、库存、物流、行为、社媒、爆款事件明细表
- DWS：库存主题、爆款主题聚合表与关键视图
- ADS：看板型 / 接口型主题表与异步物化视图

说明：

- 本版以“当前主线实现”为主，不展开历史草稿中的演示性表或重复实验对象
- `TEMPORARY TABLE / TEMPORARY VIEW` 若承担链路关键作用，也纳入字典
- 对重复出现的通用元数据字段，优先在“通用字段说明”中统一解释

## 三、通用字段说明

### 3.1 湖仓元数据字段


| 字段                               | 说明                       |
| -------------------------------- | ------------------------ |
| `_event_time` / `event_time`     | 事件发生时间，实时统计和窗口处理的时间基准    |
| `_ingest_time` / `ingest_time`   | 数据进入湖仓或进入当前层的时间          |
| `_partition_dt` / `partition_dt` | 日期分区，通常格式为 `yyyyMMdd`    |
| `_partition_hh` / `partition_hh` | 小时分区，通常格式为 `HH`          |
| `_kafka_offset`                  | Kafka 偏移量，用于审计、排查重复与丢数   |
| `_kafka_partition`               | Kafka 分区号                |
| `json_parse_status`              | JSON 解析状态，标识成功/失败或兼容解析结果 |
| `start_time`                     | 历史版本生效开始时间               |
| `end_time`                       | 历史版本失效时间                 |
| `is_current`                     | 是否当前有效版本                 |
| `version`                        | 历史版本号                    |


### 3.2 常见业务维度字段


| 字段             | 说明         |
| -------------- | ---------- |
| `sku_id`       | SKU 唯一标识   |
| `spu_code`     | SPU / 款号编码 |
| `sku_name`     | 商品名称       |
| `brand_name`   | 品牌         |
| `season`       | 季节         |

| `product_type` | 品类         |
| `store_id`     | 门店编码       |
| `warehouse_id` | 仓库编码       |

## 十四、数据血缘关系

本章用于补充一期实时湖仓的表级血缘关系，重点描述：

1. 源头业务表与事件源如何进入 ODS
2. ODS 如何构建 DIM、DWD
3. DWD 如何汇总到 DWS
4. DWS 如何服务 ADS、Dashboard 和 API

### 14.1 全链路血缘总览

```text
源头业务表 / Kafka 事件源
  ├─ ERP 主数据: M_PRODUCT / M_DIM / M_PRODUCT_ALIAS / M_ATTRIBUTESETINSTANCE / M_ATTRIBUTE
  ├─ ERP 订单: sale_order / sale_order_item
  ├─ ERP 库存: inventory_txn
  ├─ 行为流: kafka_app_click_source
  ├─ 物流流: kafka_logistics_track_source
  └─ 社媒流: kafka_social_mention_source
            │
            ▼
ODS
  ├─ cdc_m_* -> ods_m_*_append / history / latest
  ├─ cdc_sale_order + cdc_sale_order_item -> ods_order_item_cdc
  ├─ cdc_inventory_txn -> ods_inventory_txn_cdc
  ├─ kafka_app_click_source -> ods_kafka_app_click -> ods_app_click_analyze
  ├─ kafka_logistics_track_source -> ods_api_logistics_track -> ods_api_logistics_track_analyze
  └─ kafka_social_mention_source -> ods_social_mention -> ods_social_mention_analyze
            │
            ├──────────────► DIM
            │                 └─ ods_m_*_latest -> dim_sku_wide
            │
            ▼
DWD
  ├─ ods_order_item_cdc + dim_sku_wide -> dwd_order_trade_detail_rt
  ├─ ods_inventory_txn_cdc + dim_sku_wide -> dwd_inventory_flow_detail_rt
  ├─ ods_api_logistics_track_analyze + dim_sku_wide -> dwd_logistics_track_detail_rt
  ├─ ods_app_click_analyze + dim_sku_wide -> dwd_user_behavior_detail_rt
  ├─ ods_social_mention_analyze + dim_sku_wide -> dwd_social_mention_detail_rt
  └─ dwd_order_trade_detail_rt / dwd_user_behavior_detail_rt / dwd_social_mention_detail_rt
       -> dwd_hot_sku_event_detail_rt
            │
            ▼
DWS
  ├─ dwd_inventory_flow_detail_rt -> dws_inventory_atp_rt
  ├─ dwd_inventory_flow_detail_rt + dwd_order_trade_detail_rt -> dws_inventory_dispatch_rt
  ├─ dwd_hot_sku_event_detail_rt -> v_hot_sku_feature_1h -> dws_hot_sku_feature_rt
  └─ dws_hot_sku_feature_rt -> dws_hot_sku_rank_rt
            │
            ▼

  ├─ dws_inventory_atp_rt + dws_inventory_dispatch_rt -> ads_inventory_dispatch_dashboard

  └─ dws_hot_sku_feature_rt + dws_hot_sku_rank_rt -> ads_hot_sku_reorder_api
            │
            ▼

  ├─ Superset Dashboard
  ├─ Java API 服务
  └─ Doris 物化视图
```
