# 接入数据源清单

> 项目名称：服装行业实时湖仓构建项目一期  
> 文档定位：一期实时湖仓接入数据源总台账  
> 适用范围：源头系统、接入方式、增量策略、责任归属、风险管理与验收输入

---

## 一、文档目标

本清单用于统一管理一期实时湖仓接入的数据源，目标是：

1. 明确当前已接入和已规划接入的数据源范围
2. 明确每个数据源的接入方式、更新频率、增量策略和责任归属
3. 为 ODS 建设、联调验证、运维监控和项目验收提供统一台账

---

## 二、接入范围说明

一期当前重点覆盖以下类型数据源：

- ERP 关系型业务库
- 第三方物流 API
- 社媒爬虫采集数据
- 用户行为埋点数据

说明：

- 本清单优先覆盖当前一期主线已经建设或明确规划的数据源
- 后续若新增门店经营、会员、营销等主题，可继续在本清单扩展

## 六、接入方式说明

## 6.1 Flink CDC

适用对象：

- ERP 主数据
- 订单主表与明细
- 库存流水

特点：

- 增量接入能力强
- 适合关系型业务库
- 与 Flink 主链路衔接自然

## 6.2 Kafka 事件流

适用对象：

- 用户行为埋点
- 物流轨迹事件
- 社媒采集数据

特点：

- 支撑高并发实时事件输入
- 支撑重放与削峰
- 适合半结构化和多来源事件融合

## 6.3 API 拉取 + Kafka

适用对象：

- 第三方物流接口

特点：

- 先获取原始结果，再标准化入湖
- 可保留原始响应，便于排障和重放

## 6.4 爬虫采集 + Kafka

适用对象：

- 小红书等社媒平台数据

特点：

- 适合非标准 API 数据场景
- 需额外关注合规性、频控和质量治理

## 七、增量与更新策略清单


| 数据源类型   | 增量策略                 | 说明                           |
| ------- | -------------------- | ---------------------------- |
| ERP 主数据 | CDC 增量               | 支撑 append / history / latest |
| ERP 订单  | CDC 增量               | 支撑状态变化和明细更新                  |
| ERP 库存  | CDC 增量               | 支撑分钟级库存变动                    |
| 行为埋点    | 事件流增量                | 按事件时间和 Kafka 元数据入湖           |
| 物流轨迹    | API 增量拉取 + Kafka 事件化 | 以节点时间和物流单号去重                 |
| 社媒舆情    | 爬虫增量采集 + Kafka 事件化   | 以平台和内容 ID 去重                 |

## 十、接入风险与治理建议


| 风险            | 表现         | 建议                          |
| ------------- | ---------- | --------------------------- |
| 主数据字段不稳定      | 字段变更频繁     | 通过 CDC + SCD2 + latest 镜像应对 |
| 事件流 Schema 漂移 | 新增字段、字段改名  | Analyze 表延迟解析和兼容解析          |
| API 不稳定       | 超时、失败、结果缺失 | 原始响应保留、失败重试、告警              |
| 社媒采集质量波动      | 结构不稳定、值缺失  | 保留原始 JSON，独立解析状态            |
| 埋点事件质量不足      | 字段缺失、时区问题  | 加强埋点规范与 ODS 质量校验            |

5. 责任归属是否明确
